✅ 1. MÉTODOS DE NEGOCIO EN CERTIFICATE
   
   certificate.generate(template, generated_by)
   • Valida asistencia del estudiante
   • Genera código de verificación único
   • Crea PDF URL
   • Asigna fecha de expiración (365 días)
   • Registra quién generó y cuándo

   certificate.deliver(method, recipient, sent_by)
   • Simula envío (sin correo real)
   • Crea DeliveryLog automáticamente
   • Cambia certificado a status "sent"
   • Audita: método, destinatario, usuario, timestamp

   certificate.mark_as_failed(error_message, sent_by)
   • Marca como fallido
   • Registra razón del error
   • Crea log de entrega con error

   certificate.get_delivery_history()
   • Retorna todos los intentos de entrega
   • Ordenados por fecha

✅ 2. SISTEMA DE TRACKING DE ENTREGAS

   DeliveryLog se crea automáticamente con:
   • Método (email, whatsapp, link)
   • Status (success, error, pending)
   • Destinatario
   • Quién envió
   • Timestamp exacto
   • Mensajes de error

   Propiedades útiles:
   • is_successful, is_failed, is_pending
   • get_delivery_icon() - emoji para método
   • get_status_icon() - emoji para status
   • Historial completo auditable

✅ 3. ACCIONES EN DJANGO ADMIN

   Certificate Admin Actions:
   ┌─────────────────────────────────────────┐
   │ ✅ Generate Certificates (batch)        │
   │ 📧 Deliver Certificates (via email)     │
   │ ❌ Mark as Failed (con error log)       │
   │ ↩️  Reset to Pending (para reintentar)   │
   └─────────────────────────────────────────┘

   Validaciones Integradas:
   • No genera si no hay asistencia
   • No envía si no está generado
   • Verifica inscripción antes de generar
   • Auditoria: quién, qué, cuándo

   DeliveryLog Admin Actions:
   ┌─────────────────────────────────────────┐
   │ ↩️  Retry Failed Deliveries              │
   │ ✅ Mark as Successful (manual)           │
   └─────────────────────────────────────────┘

✅ 4. VISUALIZACIÓN MEJORADA EN ADMIN

   Certificate Listado:
   • Estudiante + Evento
   • Código verificación (truncado)
   • Status con badge de color
   • Última entrega (método + status)
   • Fecha emisión

   Certificate Detalle:
   • Información completa del certificado
   • Datos del estudiante (nombre, doc, email)
   • Datos del evento (nombre, fecha, categoría)
   • Plantilla con preview de imagen
   • Sección Delivery Tracking (colapsible)
     └─ Historial completo de entregas
     └─ Iconos: ✅ ❌ ⏳
     └─ Método: 📧 💬 🔗

   DeliveryLog Listado:
   • Estudiante
   • Evento (del certificado)
   • Iconos: método + status
   • Status badge
   • Fecha y hora
   • Usuario que envió

PASO 1: Ir a Certificados
┌──────────────────────────────────────────┐
│ /admin/certificados/certificate/         │
│ Ver lista de certificados (status pending)│
└──────────────────────────────────────────┘

PASO 2: Generar Certificados
┌──────────────────────────────────────────┐
│ 1. Seleccionar certificados con status   │
│    "Pending" (checkbox)                  │
│ 2. Acción: "✅ Generate Certificates"    │
│ 3. Click: "Go"                           │
│ 4. ✅ "X certificados generados"         │
│    └─ Código único para cada uno         │
│    └─ Status: Pending → Generated        │
│    └─ Auditoria: Admin User              │
└──────────────────────────────────────────┘

   Validaciones que ocurren:
   ✓ Verifica Enrollment existe
   ✓ Verifica attendance = True
   ✓ Si falla: muestra warning
   ✓ Genera código único
   ✓ Crea PDF URL
   ✓ Asigna fecha expiración

PASO 3: Entregar Certificados
┌──────────────────────────────────────────┐
│ 1. Seleccionar certificados con status   │
│    "Generated" o "Sent"                  │
│ 2. Acción: "📧 Deliver Certificates"     │
│ 3. Click: "Go"                           │
│ 4. 📧 "X certificados entregados"        │
│    └─ Crea DeliveryLog automático        │
│    └─ Envía a email del estudiante       │
│    └─ Status: Generated → Sent           │
│    └─ Auditoria: Admin User              │
└──────────────────────────────────────────┘

   Qué ocurre internamente:
   ✓ Simula envío (sin correo real)
   ✓ Crea registro en DeliveryLog
   ✓ Guarda method: email
   ✓ Guarda recipient: email del estudiante
   ✓ Guarda status: success
   ✓ Guarda sent_by: usuario que hizo click
   ✓ Guarda sent_at: timestamp exacto

PASO 4: Ver Historial de Entregas
┌──────────────────────────────────────────┐
│ 1. Click en certificado específico       │
│ 2. Sección "Delivery Tracking"           │
│    (abrir si está colapsada)             │
│ 3. Ver lista:                            │
│    ✅ ✉️ Email → juan@example.com (15:30)  │
│    ❌ 💬 WhatsApp → error (16:00)          │
│    ⏳ 🔗 Link → pending (17:00)            │
│ 4. Click en DeliveryLog para ver detalles│
└──────────────────────────────────────────┘

PASO 5: Gestionar Entregas Fallidas
┌──────────────────────────────────────────┐
│ /admin/deliveries/deliverylog/           │
│ 1. Filtrar por Status: "Error"           │
│ 2. Seleccionar entregas fallidas         │
│ 3. Acción: "↩️  Retry Failed Deliveries"  │
│ 4. Click: "Go"                           │
│ 5. ↩️  "X entregas marcadas para retry"   │
│    └─ Status: error → pending            │
│    └─ Sistema reintentará                │
└──────────────────────────────────────────┘
