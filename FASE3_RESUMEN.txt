🔄 FLUJO AHORA
═══════════════════════════════════════════════════════════════════════════════

certificate.generate(template, generated_by)
│
├─ 1. Valida asistencia ✓
├─ 2. Genera código único ✓
├─ 3. Genera PDF REAL con reportlab ✓
│  └─ Guardado en /certificates/pdfs/
├─ 4. Asigna expiración (365 días) ✓
├─ 5. Cambia a status "generated" ✓
└─ return self

certificate.deliver(method='email', recipient=email, sent_by=user)
│
├─ 1. Valida status (must be generated/sent/failed)
├─ 2. EmailService.send_certificate() → ENVÍO REAL
│  ├─ Conecta a Gmail SMTP
│  ├─ Envía email con PDF attached
│  └─ return {'success': bool, 'message': str}
├─ 3. Crea DeliveryLog automático
│  ├─ Status: success (si envío OK)
│  └─ Status: error (si envío falla)
├─ 4. Actualiza certificate.status
│  ├─ Si success → "sent"
│  └─ Si error → "failed"
└─ return delivery_log

certificate.deliver(method='whatsapp', recipient=phone, sent_by=user)
│
├─ 1. Valida status (must be generated/sent/failed)
├─ 2. WhatsAppService.send_certificate() → ENVÍO REAL
│  ├─ Conecta a Twilio API
│  ├─ Envía mensaje de WhatsApp
│  └─ return {'success': bool, 'message': str, 'sid': str}
├─ 3. Crea DeliveryLog automático
│  ├─ delivery_method: 'whatsapp'
│  ├─ Status: success/error
│  └─ recipient: número de teléfono
├─ 4. Actualiza certificate.status
└─ return delivery_log

certificate.deliver(method='link')
│
├─ 1. Valida status
├─ 2. Sin envío real (solo marca como entregado)
├─ 3. Crea DeliveryLog
├─ 4. Status → "sent"
└─ return delivery_log

🖱️ ACCIONES EN ADMIN
═══════════════════════════════════════════════════════════════════════════════

/admin/certificados/certificate/

1. ✅ Generate Certificates
   ├─ Selecciona: status = pending
   ├─ Genera PDF REAL con reportlab
   ├─ Asigna código único
   ├─ Status: pending → generated
   └─ Resultado: "X certificados generados"

2. 📧 Deliver via Email (REAL EMAIL)
   ├─ Selecciona: status = generated/sent
   ├─ Envía email REAL vía Gmail SMTP
   ├─ Adjunta PDF
   ├─ Crea DeliveryLog
   ├─ Status: generated → sent (si OK)
   └─ Resultado: "X certificados entregados via EMAIL"

3. 💬 Deliver via WhatsApp (REAL WHATSAPP)
   ├─ Selecciona: status = generated/sent
   ├─ Envía WhatsApp REAL vía Twilio
   ├─ Valida número de teléfono
   ├─ Crea DeliveryLog
   ├─ Status: generated → sent (si OK)
   └─ Resultado: "X certificados entregados via WhatsApp"

4. 🔗 Mark as Delivered (Direct Link)
   ├─ Selecciona: status = generated/sent
   ├─ Sin envío real
   ├─ Solo marca como entregado
   ├─ Status: generated → sent
   └─ Resultado: "X certificados marcados como delivered"

5. ❌ Mark as Failed
   ├─ Selecciona: status = generated/sent/failed
   ├─ Status: → failed
   ├─ Crea DeliveryLog con error
   └─ Para reintentar: Reset to Pending

6. ↩️  Reset to Pending (for retry)
   ├─ Cualquier status
   ├─ Vuelve a pending
   ├─ Mantiene código verificación
   └─ Para reintentar workflow
